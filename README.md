# PillBox IoT 💊 — Умная таблетница на базе ESP32-C3

Инновационное IoT-устройство для контроля регулярности приема лекарств. Таблетница автоматически напоминает о
необходимости приема звуковым сигналом, отправляет уведомления в Telegram и позволяет удаленно настраивать расписание
через чат-бота.

Проект разработан с учетом сетевых ограничений (работает в РФ без VPN через Google-прокси шлюз) и оптимизирован для
работы от сети (USB/кабель).

---

## 🚀 Основные возможности

* **Локальная настройка (Captive Portal):** При первом запуске устройство создает собственную Wi-Fi сеть
  `PillBox_Setup`. Настройка параметров (Wi-Fi, Chat ID, время) происходит в красивом веб-интерфейсе, который
  автоматически открывается на телефоне.
* **Синхронизация времени:** Точное время автоматически подтягивается из интернета по протоколу NTP при подключении к
  сети.
* **Уведомления в Telegram:** Устройство присылает сообщение `🔔 ВРЕМЯ ПИТЬ ТАБЛЕТКИ!`, а также отчет
  `✅ Таблетки приняты`, когда вы открываете крышку.
* **Удаленное управление:** Возможность менять время будильников прямо из чата Telegram с помощью команд `/t1 HH:MM` и
  `/t2 HH:MM`.
* **Энергонезависимая память:** Все настройки (Wi-Fi, будильники, Chat ID) сохраняются во встроенную память ESP32 (
  `Preferences`) и не стираются при отключении питания.
* **Мгновенный сброс:** Физический датчик на крышке мгновенно отключает пищалку при открытии, не дожидаясь окончания
  отправки сетевых запросов.

## 🔌 Схема подключения (Hardware)

Устройство спроектировано на базе микроконтроллера **ESP32-C3 Mini**.

| Компонент                      | Пин на ESP32-C3 | Описание подключения                                            |
|:-------------------------------|:----------------|:----------------------------------------------------------------|
| **Активный зуммер (TMB12A05)** | `GPIO 0`        | Плюс (+) -> через резистор 100 Ом к GPIO 0. Минус (-) -> к GND. || **Датчик крышки (Кнопка)    ** | `GPIO 4` | Подключение с внешним стягивающим резистором (External Pull-down). Один контакт кнопки к 3.3V, второй контакт к GPIO 4 и через резистор 10 кОм к GND. |

### Логика работы датчика крышки:

* **Крышка ЗАКРЫТА (кнопка нажата):** Контакты замкнуты на 3.3V. На пине 4 уровень **HIGH (1)**.
* **Крышка ОТКРЫТА (кнопка отпущена):** Контакты разомкнуты, пин притянут резистором к земле. На пине 4 уровень **LOW (
  0)**.

## 🛠 Инструкция по развертыванию (Setup Guide)

### Шаг 1: Подготовка Telegram-бота

1. Найдите в Telegram бота `@BotFather` и создайте нового бота с помощью команды `/newbot`. Скопируйте полученный *
   *Token**.
2. Найдите бота `@myidbot` и отправьте команду `/getid`, чтобы узнать ваш числовой **Chat ID**.
3. Отправьте вашему созданному боту команду `/start`, чтобы разрешить ему писать вам.

### Шаг 2: Настройка Google Apps Script (Прокси-шлюз)

Для стабильной отправки сообщений из РФ используется бесплатный веб-интерфейс Google в качестве моста:

1. Перейдите на [script.google.com](https://script.google.com/) и создайте новый проект.
2. Вставьте следующий код:
   ```javascript
   function doGet(e) {
     var token = e.parameter.token;
     var method = e.parameter.method || "sendMessage";
     var url = "https://api.telegram.org/bot" + token + "/" + method;
     
     var params = [];
     for (var p in e.parameter) {
       if (p !== 'token' && p !== 'method') {
         params.push(p + "=" + encodeURIComponent(e.parameter[p]));
       }
     }
     params.push("nocache=" + Math.random());
     if (params.length > 0) url += "?" + params.join("&");
     
     var response = UrlFetchApp.fetch(url, {muteHttpExceptions: true});
     return ContentService.createTextOutput(response.getContentText());
   }
   ```
3. Нажмите **«Начать развертывание»** -> **«Новое развертывание»**.
4. Выберите тип: **«Веб-приложение»**. Доступ: **«Все» (Anyone)**.
5. Скопируйте полученный URL (он понадобится для прошивки).

### Шаг 3: Настройка проекта в IDE (PlatformIO)

1. Откройте проект в CLion / VS Code с установленным плагином PlatformIO.
2. Приведите файл `platformio.ini` к следующему виду:
   ```ini
   [env:esp32-c3-devkitm-1]
   platform = espressif32
   board = esp32-c3-devkitm-1
   framework = arduino
   monitor_speed = 115200
   build_flags = 
       -D ARDUINO_USB_MODE=1
       -D ARDUINO_USB_CDC_ON_BOOT=1
   lib_deps = 
       bblanchon/ArduinoJson @ ^6.21.3
   ```
3. В файле `src/main.cpp` найдите блок констант и вставьте ваши данные:
   ```cpp
   const String BOT_TOKEN = "ВАШ_ТОКЕН_ОТ_BOTFATHER";
   const String G_SCRIPT_URL = "ВАШ_URL_ОТ_GOOGLE_SCRIPT";
   ```
4. Загрузите прошивку в плату.

## 📱 Первый запуск и настройка

1. Зайти в Телеграм-бота @IoTRegApiBot и нажать "Старт" (/start).
2. Подайте питание на таблетницу.
3. С телефона подключитесь к появившейся Wi-Fi сети `PillBox_Setup` (пароль не требуется).
4. В открывшемся окне авторизации (или перейдя по адресу `192.168.4.1`) введите:
    * **Имя Wi-Fi:** Название точки доступа вашего телефона (или домашнего роутера).
    * **Пароль Wi-Fi:** Пароль от этой сети.
    * **Chat ID:** Ваш Telegram ID (полученный на Шаге 1).
    * **Утро / Вечер:** Желаемое время напоминаний.
5. Нажмите **«Сохранить»**. Устройство перезагрузится, подключится к вашей сети и пришлет в Telegram сообщение:
   `Таблетница в сети! 🟢`.

## 🤖 Управление через Telegram

Вы можете отправлять боту следующие команды:

* `/status` — Возвращает текущее системное время таблетницы, расписание будильников и физическое состояние крышки (
  открыта/закрыта).
* `/t1 HH:MM` — Изменить время утреннего приема (например, `/t1 08:30`).
* `/t2 HH:MM` — Изменить время вечернего приема (например, `/t2 21:15`).

## 🛠 Технологический стек

* **Платформа:** ESP32-C3 (RISC-V 32-bit CPU, Wi-Fi & Bluetooth).
* **Среда разработки:** CLion / VS Code + PlatformIO.
* **Фреймворк:** Arduino (C++).
* **Многозадачность:** FreeRTOS (управление пищалкой выделено в отдельную низкоуровневую задачу для стабильной работы
  звука).
* **Сетевые протоколы:** HTTPS, DNS (для Captive Portal), NTP (синхронизация времени).
* **Облачные технологии:** Google Apps Script (проксирование трафика).
