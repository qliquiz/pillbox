### Раздел 3. Способы взаимодействия модулей системы

Разработанная система представляет собой распределенный аппаратно-программный комплекс,
состоящий из четырех основных взаимодействующих модулей:

1. **Аппаратный модуль (микроконтроллер ESP32-C3)** — конечное устройство с датчиками.
2. **Пользовательское устройство (смартфон)** — средство локальной настройки и отображения интерфейса.
3. **Облачный прокси-сервер (Google Apps Script)** — шлюз для обхода сетевых ограничений.
4. **Сервер обмена сообщениями (Telegram Bot API)** — среда удаленного управления и оповещения.

Взаимодействие между данными модулями построено на базе различных программных интерфейсов и протоколов передачи данных.
Архитектуру взаимодействия можно разделить на три ключевых сценария.

#### 3.1. Локальное взаимодействие: Смартфон ↔ ESP32-C3 (Этап настройки)

На этапе первоначальной конфигурации при отсутствии подключения к сети Интернет
взаимодействие модулей осуществляется напрямую без участия сторонних серверов.

* **Используемый интерфейс:** Локальный HTTP API.
* **Механизм взаимодействия:**
  Смартфон пользователя подключается к Wi-Fi сети микроконтроллера.
  При попытке любого сетевого запроса срабатывает встроенный DNS-перехватчик,
  перенаправляющий пользователя на веб-страницу настройки, хранящуюся в постоянной памяти ESP32-C3.
  Передача параметров осуществляется посредством отправки асинхронного HTTP-запроса методом `GET`
  с параметрами в строке запроса:
  `GET /save?s=[SSID]&p=[password]&c=[Chat_ID]&t1=[HH:MM]&t2=[HH:MM]`
  Микроконтроллер парсит строку запроса, сохраняет полученные данные в энергонезависимую память
  и выполняет перезагрузку для перехода в рабочий режим.

#### 3.2. Исходящее взаимодействие: ESP32-C3 → Google Cloud → Telegram

Для отправки уведомлений конечное устройство должно передать данные на серверы Telegram.
Из-за блокировок прямого доступа к API Telegram на территории РФ используется промежуточный облачный шлюз.

* **Используемый интерфейс:** Web API с проксированием.
* **Механизм взаимодействия:**
  ESP32-C3 формирует защищенное HTTPS-соединение с сервером Google Apps Script,
  передавая параметры сообщения в зашифрованном GET-запросе:
  `GET /macros/s/[Script_ID]/exec?token=[Bot_Token]&method=sendMessage&chat_id=[Chat_ID]&text=[Message]`
  Облачный скрипт Google принимает этот запрос, декодирует параметры и инициирует
  внутренний запрос со стороны серверов Google к официальному серверу Telegram:
  `POST https://api.telegram.org/bot[Bot_Token]/sendMessage`
  Telegram доставляет push-уведомление на смартфон пользователя в чат-бот.

#### 3.3. Входящее взаимодействие: Смартфон → Telegram → Google Cloud → ESP32-C3

Для удаленного изменения времени будильников или запроса статуса устройства пользователь
отправляет текстовые команды в чат-бот на своем смартфоне.

* **Используемый интерфейс:** Web API Telegram.
* **Механизм взаимодействия:**
    1. Команда пользователя поступает на серверы Telegram и сохраняется в очереди сообщений бота.
    2. Микроконтроллер ESP32-C3, находясь в рабочем режиме, каждые 7 секунд отправляет через Google-шлюз запрос на
       получение обновлений:
       `GET /exec?token=[Bot_Token]&method=getUpdates&offset=[Last_Update_ID+1]`
    3. Google-шлюз транслирует этот запрос в Telegram, получает массив новых сообщений в формате JSON и пересылает его
       обратно на ESP32-C3.
    4. Микроконтроллер разбирает полученный JSON-пакет с помощью библиотеки `ArduinoJson`.
       Если ID отправителя совпадает с сохраненным в памяти `Chat_ID`, устройство извлекает текст команды,
       обновляет переменные времени в оперативной памяти, перезаписывает их во Flash-память и
       отправляет пользователю ответное подтверждение об успешном изменении настроек.
